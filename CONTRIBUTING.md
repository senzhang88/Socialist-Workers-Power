# 贡献指南

> 感谢你对 Socialist-Workers-Power 项目的关注！本指南帮助你快速了解如何参与贡献。

---

## 🎯 项目愿景

我们致力于构建一个**专业、实用、可信赖**的劳动者维权知识库。

**贡献原则：**
- **准确优先**：法律信息必须准确，引用来源必须权威
- **实用导向**：不只是法律条文，更要告诉用户"怎么做"
- **保护隐私**：案例必须脱敏，不暴露个人和企业敏感信息
- **开放协作**：欢迎各种形式的贡献，尊重不同观点

## 🌟 如何贡献

### 方式1：提交 Issue（最简单）

发现错误或有建议？直接提交 Issue：

1. 访问 [Issues页面](https://github.com/senzhang88/Socialist-Workers-Power/issues)
2. 点击 "New Issue"
3. 选择对应的模板：
   - 🐛 **错误报告**（发现法律错误、数据错误）
   - 💡 **功能建议**（新增内容、改进建议）
   - 📚 **案例分享**（分享维权经历，需脱敏）
4. 详细描述问题或建议

**Issue 标签说明：**
- `bug` - 内容错误
- `enhancement` - 改进建议
- `documentation` - 文档相关
- `good first issue` - 适合新手参与
- `help wanted` - 需要帮助
- `data-update` - 数据需要更新

### 方式2：提交 Pull Request（代码/文档贡献）

想要直接修改内容？提交 PR：

```bash
# 1. Fork 本仓库（在GitHub网页上点击 Fork 按钮）

# 2. 克隆你的 Fork
$ git clone https://github.com/你的用户名/Socialist-Workers-Power.git
$ cd Socialist-Workers-Power

# 3. 创建分支
$ git checkout -b fix/具体修复内容
# 或
$ git checkout -b feature/新增功能

# 4. 修改内容
# 使用你喜欢的编辑器修改文档

# 5. 提交更改
$ git add .
$ git commit -m "fix: 修正2024年社平工资数据为12,434元"
# 或
$ git commit -m "feat: 增加工伤认定专题文档"

# 6. 推送到你的 Fork
$ git push origin fix/具体修复内容

# 7. 在GitHub上提交 Pull Request
# 访问你的 Fork 页面，点击 "Compare & pull request"
```

**PR 标题规范：**
- `fix:` 开头 - 修复错误
- `feat:` 开头 - 新增功能/文档
- `docs:` 开头 - 文档改进
- `data:` 开头 - 数据更新
- `refactor:` 开头 - 重构

## 📋 内容贡献规范

### 法律信息准确性

**必须遵守：**
- [ ] 引用法律法规必须标注具体条款（如：《劳动合同法》第47条）
- [ ] 数据必须注明来源和更新时间（如：上海人社局2025年6月发布）
- [ ] 案例必须脱敏，不包含真实人名、公司名
- [ ] 不确定的内容标注"待核实"或咨询专业律师

**推荐来源（按优先级）：**
1. 国家法律法规数据库 (flk.npc.gov.cn)
2. 上海市人力资源和社会保障局官网 (rsj.sh.gov.cn)
3. 最高人民法院司法解释
4. 各地裁判文书网判例
5. 权威法律数据库（威科先行、北大法宝）

### 文档格式规范

**Markdown 格式：**
```markdown
# 一级标题（文档标题）

## 二级标题（章节）

### 三级标题（小节）

**重点内容** 使用加粗

| 表格 | 表头 |
|------|------|
| 内容 | 内容 |

- [ ] 清单使用复选框
```

**文件命名：**
- 使用小写字母和连字符
- 示例：`job-offer-checklist.md`、`shanghai-regulations.md`

**章节结构：**
```
1. 问题定义（是什么）
2. 法律依据（为什么）
3. 应对策略（怎么做）
4. 实操步骤（具体行动）
5. 常见问题（FAQ）
6. 工具/模板（资源）
```

### 案例贡献规范

**案例必须脱敏：**

❌ **不允许：**
- 真实公司全称（如：上海某某科技有限公司）
- 真实人名（如：某某）
- 具体日期（如：2025年1月15日）→ 改为"2025年1月"
- 精确金额（如：月薪8,300元）→ 改为"月薪约8,000元"
- 可识别信息（工号、部门名称、项目代号）

✅ **允许：**
- 行业类型（如：互联网公司、制造业）
- 岗位类型（如：技术岗、行政岗）
- 大致工作年限（如：工作3年、入职2年）
- 模糊化金额（如：月薪8,000元左右）

**案例格式模板：**
```markdown
## 案例：软裁员应对

**背景：**
- 行业：互联网
- 岗位：技术岗
- 工作年限：约3年
- 月薪：约10,000元

**事件经过：**
1. 20XX年X月，公司口头通知停岗
2. ...（脱敏后的过程描述）

**应对策略：**
- ...（采取的行动）

**结果：**
- ...（最终 outcome）

**经验总结：**
- ...（可复用的经验）
```

## ❓ 常见问题

**Q：我不是程序员，可以贡献吗？**
> 当然可以！你可以：
> - 提交 Issue 指出错误
> - 在在线文档评论提出改进建议
> - 分享脱敏后的维权经历
> - 帮助审阅文档的准确性

**Q：我不确定法律信息是否正确，怎么办？**
> 标记为"待核实"，或提交 Issue 询问。我们会组织讨论确认。

**Q：我的贡献会被如何使用？**
> 所有贡献默认采用 MIT 协议开源。你的名字会出现在贡献者名单中。

**Q：公司有法务背景，可以贡献吗？**
> 非常欢迎！你的专业知识可以帮助确保内容准确性。
